# Part 5 / Clip 1: CLAUDE.md — AI에게 업무 매뉴얼 쓰기

> 강의 영상: Part 5 / Ch 01 / Clip 1 (~20분)
> 만드는 것: 본인 학습 워크스페이스용 `CLAUDE.md`  
> `~/fastcampus-cc/50-my-work/Part05-뜯어보기/my-cc-study/CLAUDE.md`

---

## 이 클립에서 만드는 것

지금까지 강의 워크스페이스에서 실습을 해왔습니다. 돌아보면 이상할 정도로 자동으로 처리된 것들이 많았죠. 클립을 끝내고 `완료`만 입력하면 다음 클립이 안내됐고, 진도도 자동으로 체크됐습니다. 새 실습을 시작하면 결과물 폴더가 생기고, 필요한 파일도 준비돼 있었습니다. 답변은 늘 한국어로 나왔고, 결과물은 정해진 위치에 저장됐고요.

이 동작은 대부분 `CLAUDE.md`라는 파일에서 시작됩니다. 워크스페이스 루트에 놓는 마크다운 파일입니다. 클로드코드는 새 세션을 열 때 이 파일을 읽고, 거기에 적힌 규칙을 작업 기준으로 삼습니다.

오늘은 본인 학습 워크스페이스용 `CLAUDE.md`를 만듭니다. 강의 워크스페이스에 있는 파일은 강의 진행용이라 건드리지 않습니다. 별도로 받은 공부용 워크스페이스 안에 내 규칙 파일을 작성합니다.

작업 전과 후의 차이는 이렇습니다.

- 새 세션마다 직업, 답변 스타일, 저장 위치를 다시 말하지 않아도 됩니다.
- 강의 워크스페이스가 왜 자동으로 한국어 답변과 결과물 저장을 처리했는지 직접 확인합니다.
- 강의가 끝난 뒤에도 `my-cc-study/` 폴더를 본인 공부방으로 계속 쓸 수 있습니다.

이번 클립에서 할 일은 두 가지입니다. 먼저 강의 워크스페이스의 실제 `CLAUDE.md`를 열어 동작 방식을 확인합니다. 그다음 본인 학습 워크스페이스에 직접 `CLAUDE.md`를 한 장 작성합니다.

---

## 핵심 개념

### CLAUDE.md는 워크스페이스 약속 노트입니다

`CLAUDE.md`는 “이 폴더에서 일할 때는 이렇게 해”라고 적어두는 매뉴얼입니다. 클로드코드가 새 세션을 시작할 때마다 자동으로 읽기 때문에, 같은 지시를 매번 다시 입력할 필요가 없습니다.

둘 수 있는 위치는 두 곳입니다.

- `./CLAUDE.md` — 현재 워크스페이스에서만 적용됩니다.
- `~/.claude/CLAUDE.md` — 모든 워크스페이스에 적용됩니다.

오늘은 워크스페이스 루트에 둡니다. 경로는 `my-cc-study/CLAUDE.md`입니다. 이 파일이 있으면 `my-cc-study` 안에서 클로드코드를 켤 때마다 자동으로 적용됩니다.

### 강의 워크스페이스 CLAUDE.md의 3섹션

본인 파일을 만들기 전에 잘 만든 예시를 먼저 봅니다. 강의 워크스페이스 루트에 있는 `CLAUDE.md`를 안티그래비티 에디터에서 열어보세요. 읽기만 합니다. 이 파일을 수정하면 강의 진행이 꼬일 수 있습니다.

여기서는 세 부분만 보면 됩니다.

첫 번째는 기본 규칙입니다.

```text
- 모든 답변은 한글로 합니다
- 전문 용어는 쉬운 말로 풀어서 설명합니다
- 결과물은 눈에 보이는 형태로 만듭니다
```

강의 내내 한국어 답변이 나오고, 결과물이 눈에 보이는 형태로 만들어진 이유가 여기에 있습니다. 따로 말하지 않아도 매 세션에 적용됐던 규칙입니다.

두 번째는 자연어 트리거입니다.

```text
| `막혔어요` | `/stuck` | 에러/문제 진단 |
```

“막혔어요”라고 입력하면 `/stuck` 명령이 실행됐던 것, 기억나실 겁니다. 자연어 표현을 슬래시 명령에 연결해둔 방식입니다.

세 번째는 결과물 저장 규칙입니다.

```text
모든 실습 결과는 50-my-work/Part{NN}-제목/실습{NN}-제목/에 저장합니다.
```

AI가 폴더를 만들고 결과물을 저장했던 것도 이 규칙 때문입니다. 매번 “어디에 저장해줘”라고 덧붙이지 않아도 됐죠.

이 세 가지가 `CLAUDE.md`의 기본 뼈대입니다. 기본 규칙, 자연어 트리거, 저장 규칙. 본인 파일도 이 구조를 따라가면 됩니다.

### 두 워크스페이스 역할 분리

Part 5부터는 워크스페이스가 두 개입니다.

- 강의 워크스페이스 — `~/fastcampus-cc/`  
  강의 진행용입니다. `CLAUDE.md`나 스킬 파일을 수정하지 않습니다.

- 학습 워크스페이스 — `~/fastcampus-cc/50-my-work/Part05-뜯어보기/my-cc-study/`  
  본인 공부용입니다. 여기에는 자유롭게 파일을 만들고 수정해도 됩니다.

헷갈리기 쉬운 지점입니다. 강의 워크스페이스는 수업 진행 장치가 들어 있는 곳이고, 학습 워크스페이스는 내 작업 공간입니다. 본인 자산은 `my-cc-study/` 안에 만듭니다.

## 진행 흐름

### 1. `/part05` 스킬 호출 → 학습 워크스페이스 준비

안티그래비티에 강의 워크스페이스가 열려 있고, 클로드코드도 실행된 상태에서 시작합니다.

```text
/part05
```

스킬이 진도를 확인한 뒤 클립 선택지를 띄웁니다. 여기서 `Clip 1`을 선택합니다.

학습 워크스페이스는 스킬이 준비합니다. `cp -r` 같은 명령을 직접 입력하지 않아도 됩니다.

```text
✓ ~/fastcampus-cc/50-my-work/Part05-뜯어보기/my-cc-study/ 시드 복사 완료
✓ 50-my-work/Part05-뜯어보기/실습18-CLAUDEmd/ 진도 폴더 준비 완료
README.md  10-curriculum/  20-concepts/  30-resources/  40-ideas/  sandbox/
```

여기서 `CLAUDE.md`만 빠져 있습니다. 실수로 빠진 게 아닙니다. 이번 클립에서 직접 만들 파일이라 일부러 비워둔 상태입니다.

### 2. 강의 워크스페이스 CLAUDE.md 보기

본격 작업 전에 예시를 확인합니다. 안티그래비티 에디터에서 강의 워크스페이스 루트의 `CLAUDE.md`를 여세요. 위에서 본 기본 규칙, 자연어 트리거, 결과물 저장 규칙이 실제로 어떻게 적혀 있는지 봅니다.

다시 말하지만, 이 파일은 읽고 닫습니다. 수정하지 않습니다.

### 3. 새 창에서 학습 워크스페이스 열기

이제 학습 워크스페이스로 이동합니다. 안티그래비티에서 새 창을 열고, `Open Folder`로 아래 폴더를 선택합니다.

```text
~/fastcampus-cc/50-my-work/Part05-뜯어보기/my-cc-study/
```

새 창이 뜨면 안티그래비티 통합 터미널에서 확인합니다.

```bash
ls
```

`README.md`, `10-curriculum/`, `20-concepts/`, `30-resources/`, `40-ideas/`, `sandbox/`가 보이면 정상입니다. 아직 `CLAUDE.md`는 없어야 합니다.

### 4. `/init`으로 CLAUDE.md 초안 만들기

학습 워크스페이스 안에서 클로드코드를 실행합니다.

```bash
claude
```

여기는 강의 워크스페이스와 분리된 새 세션입니다. 클로드코드가 시작되면 `/init`을 입력합니다.

```text
/init
```

클로드코드가 1분 정도 폴더를 살펴봅니다. “학습 노트 / 자료 / 아이디어 정리 워크스페이스로 보입니다”처럼 현재 폴더를 해석하고, `CLAUDE.md` 초안을 제안합니다. 그대로 저장하면 됩니다.

안티그래비티 에디터로 돌아와 새로 생긴 `CLAUDE.md`를 열어보세요. 폴더 구조를 바탕으로 만든 초안이 들어 있을 겁니다. 다만 이 초안은 폴더만 보고 만든 것이므로, 아직 “내 작업 방식”까지 담겼다고 보기는 어렵습니다.

### 5. 인터뷰로 본인 맥락 넣기

학습 워크스페이스의 클로드코드 세션에서 이렇게 요청합니다.

```text
내가 클로드코드 공부하는 사람이라는 맥락이 더 반영되도록
CLAUDE.md를 인터뷰로 보완해줘.

한 번에 하나씩 5개 질문 후 종료.
- 내 직업/배경
- 답변 스타일 선호 (쉬운 말? 예시 많이?)
- 결과물 포맷 선호 (표/리스트/문서)
- 자주 다룰 주제
- 피해야 할 것
```

질문은 하나씩 옵니다. 답변은 길 필요 없습니다.

- 직업/배경 — 본인 직업을 한 줄로 답합니다. 가상 프로필도 괜찮습니다. 예: 마케팅 기획자, 콘텐츠 운영자, 직장인.
- 답변 스타일 — 존댓말, 쉬운 말, 예시를 많이 드는 방식처럼 선호를 말합니다.
- 결과물 포맷 — 표, 리스트, 문서, 3줄 요약 등 자주 쓰는 형태를 답합니다.
- 자주 다룰 주제 — 본인 업무나 관심 분야를 적습니다.
- 피해야 할 것 — 너무 긴 답변, 영어 전문 용어 남발, 모호한 설명처럼 싫은 방식을 말합니다.

다섯 개에 모두 답하면 클로드코드가 `CLAUDE.md`를 다듬어 저장합니다. 안티그래비티에서 다시 열어 바뀐 내용을 확인하세요.

### 6. 새 세션에서 동작 확인하기

마지막으로 실제로 반영되는지 확인합니다. 클로드코드를 한 번 종료하고 다시 켭니다.

```bash
exit
claude
```

새 세션이 시작되면 같은 질문을 던져봅니다.

```text
클로드코드를 더 공부하려면 뭐부터 보면 좋을까?
```

인터뷰에서 답한 직업, 답변 스타일, 자주 다룰 주제가 응답에 반영되어야 합니다. 질문은 같아도 답이 달라집니다. `CLAUDE.md`는 바로 이 차이를 만들기 위해 쓰는 파일입니다.

## 결과물

이번 클립이 끝나면 세 가지가 남습니다.

- `my-cc-study/CLAUDE.md` — 오늘 직접 만든 본인 맞춤 업무 매뉴얼입니다.
- `실습18-CLAUDEmd/README.md` — 스킬이 자동으로 만든 진도 메타 파일입니다.
- Before/After 응답 비교 — 학습 워크스페이스 세션에서 `CLAUDE.md` 적용 전후 차이를 확인한 기록입니다.

강의 워크스페이스로 돌아와 `완료` 또는 `/wrap`을 입력하면 스킬이 정리합니다. `progress.json`에는 `my_cc_study_initialized=true`가 기록되고, 다음 클립이 안내됩니다.

`my-cc-study/`는 강의가 끝난 뒤에도 그대로 둡니다. 다음에 안티그래비티에서 이 폴더를 열고 `claude`를 실행하면, 오늘 만든 `CLAUDE.md`가 적용된 상태로 바로 시작할 수 있습니다.

## 자주 막히는 포인트

- `/part05`가 실행되지 않음 — 강의 워크스페이스 밖에서 호출했을 가능성이 큽니다. 안티그래비티에서 `~/fastcampus-cc/`를 열고 클로드코드를 실행하세요.
- 학습 워크스페이스가 이미 있음 — 이전 시도 흔적입니다. 스킬이 `.bak.YYYYMMDD-HHMMSS` 형식으로 백업하므로 그대로 진행하면 됩니다.
- `/init` 결과가 영어로 나옴 — 클로드코드 기본값 때문일 수 있습니다. `한국어로 다시 써줘`라고 입력하세요.
- `/init`이 먹지 않음 — 클로드코드 버전이 낮을 수 있습니다. `claude --version`으로 확인한 뒤 업데이트합니다.
- 인터뷰가 2~3개 질문만 하고 끝남 — 요청을 짧게 해석한 경우입니다. `5개 질문을 모두 채워줘`라고 다시 요청하세요.
- Before/After 차이가 거의 없음 — `CLAUDE.md` 내용이 너무 넓게 적혔을 수 있습니다. `내 직업과 자주 하는 작업에 맞게 더 구체적으로 다듬어줘`라고 요청합니다.
- 인터뷰 답변에 개인정보가 걱정됨 — 가상 프로필로 답해도 됩니다. `민감 정보는 빼고 작성해줘`라고 함께 말하세요.
- 안티그래비티 새 창을 여는 방법을 모르겠음 — `File > New Window`를 사용합니다. 폴더는 `File > Open Folder`로 엽니다.
- 강의 워크스페이스 `CLAUDE.md`를 수정해버림 — 강의 파일과 본인 파일을 헷갈린 경우입니다. 강의 파일은 git에서 복원하고, 본인 파일은 학습 워크스페이스에 만듭니다.
