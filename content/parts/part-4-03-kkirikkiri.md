## 🎯 이 클립에서 만드는 것

지난 클립에서 docs-guide로 공식 문서 읽는 법을 익혔죠. 이번에는 그 docs-guide를 한 번 더 쓰는 것부터 시작합니다. 클로드코드 자체에 **Agent Teams**라는 기능이 있다는데, 이게 뭔지 docs-guide로 먼저 확인하고 들어가요.

그 다음 만나볼 친구가 **kkirikkiri (끼리끼리)** 입니다. Agent Teams를 그대로 쓰면 매번 자연어 요청을 잘 짜야 하는데, kkirikkiri는 그걸 "스킬 한 줄 호출"로 줄여줘요. 한국어 한 문장 던지면 인터뷰 두세 번 해서 팀을 짜고, 병렬로 분석하고, 리포트까지 만들어줍니다.

오늘 시연은 **코스메틱 브랜드 마케터**의 SNS 캠페인 기획이에요. 본인 직무로 바꿔서 따라 하시면 됩니다. 영업, 기획, HR, 일반 사무직 다 같은 흐름으로 동작해요.

| Before | After |
|---|---|
| AI한테 일을 시키면 한 명한테 던지듯 한 사람 답만 받음 | 본인 일을 여러 관점(기획자/실무자/분석가/AI 활용 전문가)이 동시에 분석한 리포트를 받음 |
| 클로드코드의 Agent Teams 기능이 있다는 것도 모름 | 자체 기능 + 그걸 스킬로 쉽게 쓰는 방법까지 둘 다 앎 |
| Part 6에서 만들 스킬이 어떤 형태일지 막연함 | 오늘 받은 우선순위가 그대로 Part 6 스킬 청사진이 됨 |

이번 클립 끝에는 본인이 받은 우선순위 중 1~2개를 정리합니다. 그게 Part 6에서 실제로 만들 스킬의 청사진이에요.

## 💡 핵심 개념

### Agent Teams는 클로드코드 자체 기능입니다

먼저 짚고 갈게요. kkirikkiri는 마법으로 동작하는 게 아니에요. 클로드코드 안에 이미 **Agent Teams**라는 기능이 있고, kkirikkiri는 그걸 쉽게 쓰게 만든 껍데기입니다. docs-guide로 공식 정의 가져오면 이렇게 나와요.

```text
"Coordinate multiple Claude Code instances working together as a team,
 with shared tasks, inter-agent messaging, and centralized management."

→ 여러 클로드코드 인스턴스를 팀으로 묶어 공유 태스크, 메시지,
   중앙 관리로 협업하게 하는 기능.
```

핵심 동작은 네 가지로 정리됩니다.

| 요소 | 역할 |
|---|---|
| Team Lead | 메인 세션이 팀 조직과 태스크 할당 |
| 독립 팀원들 | 각자 고유 컨텍스트 윈도우, 직접 메시지 교환 |
| 공유 태스크 리스트 | 자동 의존성 관리, race condition 방지 |
| 메시지 시스템 | 팀원 간 자동 전달 |

활성화하려면 `~/.claude/settings.json`에 환경변수 한 줄 추가하고 클로드코드를 재시작하면 됩니다. 다만 아직 **experimental** 상태예요. 동작은 하지만 공식 안정화 전 단계라 보시면 됩니다.

### kkirikkiri는 Agent Teams를 스킬로 감싼 친구입니다

Agent Teams를 직접 쓰면 매번 자연어로 "이런 팀 만들어서 이런 일 시켜줘"를 잘 적어야 해요. kkirikkiri는 그걸 한 문장 + 자동 인터뷰로 줄였습니다.

| 구분 | Agent Teams 직접 | kkirikkiri |
|---|---|---|
| 호출 | 매번 자연어 요청 직접 작성 | `/kkirikkiri 한 문장` |
| 팀 구성 | 페르소나·역할 직접 정의 | 인터뷰 2~3개 → 자동 구성 |
| 환경 스캔 | 어떤 도구 있는지 직접 파악 | 자동 (Codex / Gemini CLI / 기존 에이전트 감지) |
| 검증 | 결과 직접 점검 | 검증 루프 자동 (최대 3라운드 재시도) |
| 공유 메모리 | 직접 관리 | `.kkirikkiri/` 자동 생성·관리 |
| 재사용 | 매번 처음부터 | `.claude/agents/`에 저장 → 다음 프로젝트 재사용 |

여기서 한 번 더 짚고 갈 게 있어요. **"스킬"** 이라는 단어. Clip 1에서 플러그인 구성요소 중 하나로 등장했죠. 자동화 작업 흐름이라고 했던 거. kkirikkiri를 호출하면 `SKILL.md`에 정의된 8단계 워크플로우(인터뷰 → 환경 스캔 → 팀 구성 → 실행 → 검증 → 결과)가 자동으로 돌아갑니다.

**Part 6에서 여러분이 직접 만들 게 바로 이 형태**예요. 본인 작업 자동화를 SKILL.md로 정의하고, 자연어 한 문장으로 호출하게 만드는 거. 오늘 kkirikkiri 쓰면서 "스킬이 호출되면 이렇게 동작하는구나"를 체감하시면 됩니다.

### 팀장은 가장 좋은 모델이 맡습니다

kkirikkiri 팀에는 원칙이 하나 있어요. 팀장은 항상 가장 성능 좋은 모델(기본은 Opus)이 맡고, **계획·배분·검증만** 합니다. 코드를 직접 작성하지 않아요. 각 팀원의 역할은 엄격히 분리되고, 팀장은 결과를 통합하는 역할입니다.

이게 왜 중요하냐면, 본인이 Part 6에서 스킬 만들 때도 같은 원칙을 쓸 수 있어요. 메인 세션이 지휘하고, 무거운 분석은 서브 에이전트에 맡기는 구조.

## 🚶 진행 흐름

### 1. docs-guide로 Agent Teams 개념 확인

지난 클립에서 배운 docs-guide를 그대로 씁니다. 클로드코드 자체 기능이라 공식 문서가 가장 정확해요.

```text
/docs-guide:docs-guide Claude Code에서 Agent Teams가 뭐고, 어떻게 동작하는지 알려줘.
```

답이 오면 `agent-teams-overview.md`로 저장합니다. 핵심은 두 줄이에요. **여러 클로드코드 인스턴스를 팀으로 묶는 기능이다. 환경변수로 활성화하고 자연어로 호출한다.**

### 2. 환경변수 활성화

kkirikkiri를 쓰려면 Agent Teams 환경변수가 켜져 있어야 합니다. kkirikkiri가 친절하게 점검해줘요.

```text
kkirikkiri 환경 점검해줘.
```

미설정이면 이렇게 안내가 나옵니다.

```text
끼리끼리 환경 점검
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

필수 조건:
  ✓ Claude Code 설치됨
  ✗ Agent Teams 환경변수 미설정

    다음을 ~/.claude/settings.json에 추가하세요:
    { "env": { "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1" } }
```

`~/.claude/settings.json`을 열어서 환경변수를 추가합니다. 파일이 없으면 새로 만들어도 돼요.

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

저장하고 **클로드코드 재시작**합니다. 재시작 안 하면 환경변수가 안 잡혀요. 다시 점검 돌리면 두 줄 다 체크 표시로 바뀝니다.

```text
✓ Claude Code 설치됨
✓ Agent Teams 환경변수 설정됨
```

### 3. 본인 직무로 kkirikkiri 호출

이제 본 시연이에요. 강사는 코스메틱 브랜드 마케터로 가지만, 여러분은 본인 직무로 바꿔서 입력하세요. 핵심은 **매주 하는 일 5~7개 + 다가오는 핵심 작업 1개**를 구체적으로 적는 거예요.

```text
/kkirikkiri 

나는 코스메틱 브랜드 마케터야.
우리 브랜드는 감성·미감 요소를 강조하는 코스메틱이고,
이번에 SNS 마케팅을 본격 시작하면서 바디 미스트 신제품 캠페인을 기획하려고 해.
인스타그램이 메인 채널이 될 거야.

매주 하는 일들:
1. 신제품 캠페인 기획 (현재 진행 중인 핵심 작업)
2. 벤치마킹할 브랜드 리서치 (감성·미감 지향 코스메틱들)
3. 인스타 콘텐츠 기획·발행 (제품 촬영·스타일링·캡션)
4. 인플루언서·아티스트 협업 검토
5. 캠페인 데이터 정리·보고
6. 고객 리뷰·반응 분석
7. 월간 마케팅 리포트

이 일들에서 병목 진단 + AI 활용 가능 영역 + 우선순위를 받고 싶어.
특히 새로 시작할 SNS 캠페인 방향성에 도움 되도록.
```

kkirikkiri 스킬이 8단계 워크플로우를 시작합니다. 인터뷰 옵션이 두 번 나와요. **"디스커버리"** 와 **"제대로"** 골라주세요.

### 4. 팀 자동 구성 + 병렬 실행

옵션 골라주면 kkirikkiri가 팀을 자동으로 구성합니다. 입력 컨텍스트에 따라 팀원 구성이 매번 달라져요. 위 시나리오라면 이렇게 짭니다.

```text
[팀 자동 구성]
├── 팀장 (Lead, Opus) — 통합·우선순위 판단
├── 브랜드 전략가 (Opus) — 감성·미감 톤 분석, 차별화 포인트
├── SNS 캠페인 기획자 (Opus) — 인스타 캠페인 + 콜라보 검토
├── 데이터 분석가 (Opus) — 리뷰·반응·경쟁 모니터링 영역
└── AI 활용 전문가 (Opus) — 자동화 매핑
```

팀 구성을 보여주고 "시작할까요?" 물어보면 **"네, 시작해주세요"** 답합니다. 그러면 5~7분 정도 병렬 실행이 돌아가요. 4명이 동시에 분석하는 거라 차 한잔 마시고 오시면 됩니다.

```text
✓ 브랜드 전략가 분석 중
✓ SNS 캠페인 기획자 분석 중
✓ 데이터 분석가 분석 중
✓ AI 활용 전문가 분석 중
✓ 팀장 결과 통합 중
```

1라운드 결과 품질이 충분하면 검증 루프가 통과로 끝나고, 부족하면 자동으로 팀을 재구성합니다(최대 3라운드).

### 5. 받은 리포트 읽기

리포트가 `.kkirikkiri/reports/discovery-report.md`에 저장됩니다. 핵심 부분이 이렇게 나와요.

```text
[병목 진단]
- 벤치마킹 리서치 — 매주 8시간, 수동 사이트 방문·캡처·정리
- 인스타 콘텐츠 기획 — 캡션·해시태그 매번 처음부터 작성
- 캠페인 데이터 정리 — 인사이트 추출보다 데이터 모으기에 시간 더 씀
- 고객 리뷰 분석 — 양 많아 핵심 패턴 놓침

[AI 활용 가능 영역]
- 벤치마킹 자동 수집·요약 (매우 높음) — 매주 8시간 → 1시간
- 인스타 캡션·해시태그 초안 자동 생성 (높음)
- 캠페인 데이터 자동 집계 + 인사이트 추출 (높음)
- 고객 리뷰 감성·키워드 분석 자동화 (중간)

[새 캠페인 방향성 — 다관점 통합]
- 브랜드 전략가: 감성·미감 톤은 "한 컷의 작품" 같은 비주얼이 핵심
- SNS 캠페인 기획자: 아티스트 콜라보 + 한정판 비주얼 시리즈 추천
- 데이터 분석가: 경쟁 브랜드 콜라보 효과 평균 3배 이상
- AI 활용: 콜라보 작가·아티스트 후보 자동 리서치로 시간 절감

[우선순위]
1순위: 벤치마킹 자동 리서치 — 주 7시간 절감
2순위: 인스타 캡션·해시태그 초안 자동 생성
3순위: 캠페인 데이터 자동 집계
```

혼자 봤으면 안 보였을 패턴이 다관점에서 잡힙니다. 특히 "수집·정리에 시간이 다 쓰인다"는 진단처럼, 한 발 떨어진 시선이 묶여서 나오는 게 이 도구의 핵심 가치예요.

### 6. 1~2순위 정리 — Part 6 스킬 청사진 만들기

리포트 받았으면 마지막 단계. 우선순위 중 본인이 가장 먼저 해보고 싶은 **1~2개만** 골라서 정리합니다. 이게 Part 6에서 만들 스킬의 청사진이 돼요.

```text
방금 받은 리포트의 우선순위 1-2개만 골라서, 
이걸 클로드코드 스킬로 만들면 어떻게 동작할지 한 줄씩 정리해줘.
```

이렇게 나옵니다.

```text
1순위: 벤치마킹 자동 리서치 스킬
→ 브랜드 URL 목록 + 키워드 입력 → 매주 변경사항·신캠페인 자동 수집 → 요약 리포트

2순위: 인스타 캡션·해시태그 초안 생성 스킬
→ 제품 정보 + 톤·키워드 입력 → 캡션 3안 + 해시태그 세트 자동 생성
```

`priority-summary.md`로 저장하면, Part 6 시작할 때 이 파일을 꺼내서 그대로 스킬 제작에 들어갑니다.

| docs-guide가 Agent Teams 페이지 못 가져옴 | URL 변경 또는 일시 장애 | 클로드코드에 일반 검색으로 대체. 핵심 내용은 같음 |
| 인터뷰 옵션이 헷갈림 | 처음 사용 | "디스커버리" + "제대로" 두 개만 고르면 됨 |
| 팀 구성이 영상과 다름 | 입력 컨텍스트 차이 | 정상. 본인 입력에 맞춰 매번 다르게 구성됨 |
| 병렬 실행이 5분 이상 걸림 | 4명 동시 분석 | 정상. 차 한잔 마시며 대기 |
| 리포트가 두루뭉술 | 직무·일 설명이 모호 | "마케터" → "스타트업 코스메틱 마케터, 바디 미스트 신제품 캠페인 기획" 처럼 구체적으로 |
| 우선순위 결정이 어렵다 | 다 좋아 보임 | 기준: 주 4시간 이상 절감 + 본인이 자신 있는 영역 |
| `.kkirikkiri/` 폴더가 어디 있는지 | 공유 메모리 위치 | 워크스페이스 루트에 자동 생성. 다른 작업에는 영향 없음 |
| 본인 직무가 마케터가 아님 | 일반 적용 | 영업, 기획, HR, 일반 사무직 다 같은 흐름. 매주 일 + 다가오는 핵심 작업만 구체적으로 |
