## 이 클립에서 만드는 것

지난 클립에서는 docs-guide로 공식 문서를 읽는 법을 익혔습니다. 이번에는 그 흐름을 그대로 이어갑니다. 클로드코드에 Agent Teams라는 기능이 있는데, 먼저 docs-guide로 이 기능이 뭔지 확인합니다.

그다음 사용할 도구는 kkirikkiri(끼리끼리)입니다. Agent Teams를 직접 쓰려면 매번 “어떤 팀을 만들고 어떤 일을 맡길지” 자연어로 길게 설명해야 합니다. kkirikkiri는 이 과정을 한 줄 호출로 줄여줍니다. 한국어로 요청을 넣으면 짧게 인터뷰하고, 팀을 짜고, 병렬로 분석한 뒤 리포트까지 남깁니다.

시연 예시는 코스메틱 브랜드 마케터의 SNS 캠페인 기획입니다. 실습할 때는 본인 직무로 바꿔 넣으면 됩니다. 영업, 기획, HR, 일반 사무직도 같은 방식으로 진행할 수 있습니다.

Before:
- AI에게 일을 시키면 한 사람에게 물어본 것처럼 단일 관점 답변만 받습니다.
- 클로드코드에 Agent Teams 기능이 있는지도 모릅니다.
- Part 6에서 만들 스킬의 형태가 아직 흐릿합니다.

After:
- 기획자, 실무자, 분석가, AI 활용 전문가 관점이 섞인 리포트를 받습니다.
- 클로드코드 기본 기능과, 그 기능을 스킬로 쉽게 쓰는 방법을 함께 압니다.
- 오늘 정한 우선순위가 Part 6 스킬의 초안이 됩니다.

클립이 끝나면 리포트에서 1~2순위만 골라 정리합니다. 그 파일을 Part 6에서 다시 꺼내 씁니다.

## 핵심 개념

### Agent Teams는 클로드코드 자체 기능입니다

먼저 구분해야 합니다. kkirikkiri가 팀 기능을 새로 만든 것은 아닙니다. 클로드코드 안에 Agent Teams라는 기능이 있고, kkirikkiri는 그 기능을 쓰기 쉽게 감싼 스킬입니다.

docs-guide로 공식 정의를 가져오면 이런 설명이 나옵니다.

```text
"Coordinate multiple Claude Code instances working together as a team,
 with shared tasks, inter-agent messaging, and centralized management."

→ 여러 클로드코드 인스턴스를 팀으로 묶어 공유 태스크, 메시지,
   중앙 관리로 협업하게 하는 기능.
```

동작은 네 가지로 보면 됩니다.

- Team Lead: 메인 세션이 팀을 만들고 작업을 나눕니다.
- 독립 팀원들: 각 팀원이 자기 컨텍스트 윈도우를 갖고 움직입니다.
- 공유 태스크 리스트: 작업 의존성을 관리하고 충돌을 줄입니다.
- 메시지 시스템: 팀원 사이의 전달 내용을 자동으로 이어줍니다.

Agent Teams를 쓰려면 `~/.claude/settings.json`에 환경변수 한 줄을 넣고 클로드코드를 다시 시작해야 합니다. 아직 experimental 상태라, 기능은 쓸 수 있지만 공식 안정화 전 단계로 보면 됩니다.

### kkirikkiri는 Agent Teams를 스킬로 감싼 도구입니다

Agent Teams를 직접 쓰려면 요청문을 잘 써야 합니다. 어떤 역할이 필요하고, 누가 무엇을 분석해야 하며, 결과를 어떻게 검증할지까지 직접 설명해야 하니까요.

kkirikkiri는 그 과정을 줄여줍니다.

- 호출: 직접 요청문을 길게 쓰는 대신 `/kkirikkiri 한 문장`으로 시작합니다.
- 팀 구성: 역할과 페르소나를 직접 정하지 않아도, 인터뷰 2~3개를 거쳐 자동으로 잡아줍니다.
- 환경 스캔: Codex, Gemini CLI, 기존 에이전트 같은 사용 가능한 도구를 확인합니다.
- 검증: 결과가 부족하면 최대 3라운드까지 다시 시도합니다.
- 공유 메모리: `.kkirikkiri/` 폴더를 만들고 진행 상황을 저장합니다.
- 재사용: `.claude/agents/`에 저장해 다음 프로젝트에서도 꺼내 쓸 수 있습니다.

여기서 “스킬”이라는 단어를 다시 봐야 합니다. Clip 1에서 플러그인 구성요소 중 하나로 다뤘던 그 스킬입니다. 자동화된 작업 흐름이라고 설명했죠.

kkirikkiri를 호출하면 `SKILL.md`에 적힌 워크플로우가 실행됩니다. 인터뷰하고, 환경을 확인하고, 팀을 만들고, 실행하고, 검증하고, 결과를 저장하는 흐름입니다.

Part 6에서 직접 만들 것도 이와 같은 형태입니다. 본인 작업을 `SKILL.md`로 정의하고, 자연어 한 문장으로 호출하게 만드는 것. 오늘 실습은 “스킬이 호출되면 실제로 어떻게 움직이는지” 보는 시간이기도 합니다.

### 팀장은 계획과 검증만 맡습니다

kkirikkiri 팀에서는 팀장이 계획, 배분, 검증을 맡습니다. 기본값은 가장 성능 좋은 모델입니다. 팀장이 직접 결과물을 다 쓰는 구조가 아니라, 각 팀원이 맡은 관점에서 분석하고 팀장이 그 결과를 합칩니다.

이 구조는 Part 6에서도 그대로 참고할 수 있습니다. 메인 세션은 지휘하고, 무거운 분석은 서브 에이전트에게 맡기는 방식입니다.

## 진행 흐름

### 1. docs-guide로 Agent Teams 개념 확인

지난 클립에서 쓴 docs-guide를 그대로 사용합니다. 클로드코드 자체 기능이므로 공식 문서를 먼저 보는 게 맞습니다.

```text
/docs-guide:docs-guide Claude Code에서 Agent Teams가 뭐고, 어떻게 동작하는지 알려줘.
```

답변이 오면 `agent-teams-overview.md`로 저장합니다.

여기서 잡아야 할 내용은 두 줄입니다. Agent Teams는 여러 클로드코드 인스턴스를 팀으로 묶는 기능입니다. 환경변수로 켜고 자연어로 호출합니다.

### 2. 환경변수 활성화

kkirikkiri를 쓰려면 Agent Teams 환경변수가 켜져 있어야 합니다. 먼저 점검을 돌립니다.

```text
kkirikkiri 환경 점검해줘.
```

설정이 빠져 있으면 이런 안내가 나옵니다.

```text
끼리끼리 환경 점검
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

필수 조건:
  ✓ Claude Code 설치됨
  ✗ Agent Teams 환경변수 미설정

    다음을 ~/.claude/settings.json에 추가하세요:
    { "env": { "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1" } }
```

`~/.claude/settings.json`을 열고 환경변수를 추가합니다. 파일이 없으면 새로 만들면 됩니다.

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

저장한 뒤 클로드코드를 다시 시작합니다. 재시작하지 않으면 환경변수가 잡히지 않습니다.

다시 점검하면 두 항목이 모두 통과해야 합니다.

```text
✓ Claude Code 설치됨
✓ Agent Teams 환경변수 설정됨
```

### 3. 본인 직무로 kkirikkiri 호출

이제 본 실습입니다. 강사는 코스메틱 브랜드 마케터 예시로 진행하지만, 여러분은 자기 직무를 넣으면 됩니다.

중요한 건 두 가지입니다. 매주 하는 일 5~7개를 적고, 지금 다가오는 큰 작업 1개를 구체적으로 적습니다.

```text
/kkirikkiri 

나는 코스메틱 브랜드 마케터야.
우리 브랜드는 감성·미감 요소를 강조하는 코스메틱이고,
이번에 SNS 마케팅을 본격 시작하면서 바디 미스트 신제품 캠페인을 기획하려고 해.
인스타그램이 메인 채널이 될 거야.

매주 하는 일들:
1. 신제품 캠페인 기획 (현재 진행 중인 핵심 작업)
2. 벤치마킹할 브랜드 리서치 (감성·미감 지향 코스메틱들)
3. 인스타 콘텐츠 기획·발행 (제품 촬영·스타일링·캡션)
4. 인플루언서·아티스트 협업 검토
5. 캠페인 데이터 정리·보고
6. 고객 리뷰·반응 분석
7. 월간 마케팅 리포트

이 일들에서 병목 진단 + AI 활용 가능 영역 + 우선순위를 받고 싶어.
특히 새로 시작할 SNS 캠페인 방향성에 도움 되도록.
```

kkirikkiri가 워크플로우를 시작합니다. 중간에 인터뷰 옵션이 두 번 나오면 “디스커버리”와 “제대로”를 고릅니다.

### 4. 팀 자동 구성과 병렬 실행

옵션을 고르면 kkirikkiri가 입력 내용을 보고 팀을 구성합니다. 같은 명령을 써도 직무와 상황에 따라 팀은 달라집니다.

위 예시라면 대략 이런 구성이 나옵니다.

```text
[팀 자동 구성]
├── 팀장 (Lead, Opus) — 통합·우선순위 판단
├── 브랜드 전략가 (Opus) — 감성·미감 톤 분석, 차별화 포인트
├── SNS 캠페인 기획자 (Opus) — 인스타 캠페인 + 콜라보 검토
├── 데이터 분석가 (Opus) — 리뷰·반응·경쟁 모니터링 영역
└── AI 활용 전문가 (Opus) — 자동화 매핑
```

팀 구성을 보여주고 “시작할까요?”라고 물으면 이렇게 답합니다.

```text
네, 시작해주세요
```

이후 5~7분 정도 병렬 실행이 진행됩니다. 여러 관점의 분석이 동시에 돌아가므로 조금 기다리면 됩니다.

```text
✓ 브랜드 전략가 분석 중
✓ SNS 캠페인 기획자 분석 중
✓ 데이터 분석가 분석 중
✓ AI 활용 전문가 분석 중
✓ 팀장 결과 통합 중
```

1라운드 결과가 충분하면 검증을 통과하고 끝납니다. 부족하면 팀을 다시 구성해 최대 3라운드까지 보완합니다.

### 5. 리포트 읽기

리포트는 `.kkirikkiri/reports/discovery-report.md`에 저장됩니다. 예시 결과는 이런 식입니다.

```text
[병목 진단]
- 벤치마킹 리서치 — 매주 8시간, 수동 사이트 방문·캡처·정리
- 인스타 콘텐츠 기획 — 캡션·해시태그 매번 처음부터 작성
- 캠페인 데이터 정리 — 인사이트 추출보다 데이터 모으기에 시간 더 씀
- 고객 리뷰 분석 — 양 많아 핵심 패턴 놓침

[AI 활용 가능 영역]
- 벤치마킹 자동 수집·요약 (매우 높음) — 매주 8시간 → 1시간
- 인스타 캡션·해시태그 초안 자동 생성 (높음)
- 캠페인 데이터 자동 집계 + 인사이트 추출 (높음)
- 고객 리뷰 감성·키워드 분석 자동화 (중간)

[새 캠페인 방향성 — 다관점 통합]
- 브랜드 전략가: 감성·미감 톤은 "한 컷의 작품" 같은 비주얼이 핵심
- SNS 캠페인 기획자: 아티스트 콜라보 + 한정판 비주얼 시리즈 추천
- 데이터 분석가: 경쟁 브랜드 콜라보 효과 평균 3배 이상
- AI 활용: 콜라보 작가·아티스트 후보 자동 리서치로 시간 절감

[우선순위]
1순위: 벤치마킹 자동 리서치 — 주 7시간 절감
2순위: 인스타 캡션·해시태그 초안 자동 생성
3순위: 캠페인 데이터 자동 집계
```

혼자 보면 놓치기 쉬운 부분이 여기서 잡힙니다. 예를 들어 “수집과 정리에 시간이 너무 많이 간다”는 진단은 당사자일수록 잘 못 봅니다. 여러 관점이 붙으면 그런 병목이 더 또렷하게 드러납니다.

### 6. 1~2순위 정리하기

리포트를 받았으면 마지막으로 우선순위 중 1~2개만 고릅니다. Part 6에서 실제로 스킬로 만들 후보입니다.

```text
방금 받은 리포트의 우선순위 1-2개만 골라서, 
이걸 클로드코드 스킬로 만들면 어떻게 동작할지 한 줄씩 정리해줘.
```

결과는 이렇게 정리할 수 있습니다.

```text
1순위: 벤치마킹 자동 리서치 스킬
→ 브랜드 URL 목록 + 키워드 입력 → 매주 변경사항·신캠페인 자동 수집 → 요약 리포트

2순위: 인스타 캡션·해시태그 초안 생성 스킬
→ 제품 정보 + 톤·키워드 입력 → 캡션 3안 + 해시태그 세트 자동 생성
```

이 내용을 `priority-summary.md`로 저장합니다. Part 6을 시작할 때 이 파일을 보고 바로 스킬 제작으로 넘어갑니다.
