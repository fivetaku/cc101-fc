# Part 6 / Clip 3: deep-research v1 분석·비교·개선 → v2

> 강의 영상: Part 6 / Ch 01 / Clip 3 (~20분)
> 만드는 것: 본인 v1을 풀스펙과 같은 폴더에 모아 비교 → SKILL.md 직접 손으로 디벨롭한 v2
> 시연 주제: 같은 김밥 맛집 (v1 ↔ v2 결과 비교)

---

## 이 클립에서 만드는 것

Clip 2에서 만든 `deep-research` v1이 있습니다. 이번 클립에서는 그 v1을 같은 폴더에 풀스펙 스킬과 모아놓고 비교한 뒤, 부족한 부분 1~2가지를 본인 손으로 SKILL.md에 추가해 **v2**로 진화시킵니다.

핵심은 결과물보다 사이클입니다. **스킬은 한 번 만들면 끝이 아니라 라이프사이클**이라는 메시지를 손으로 한 번 굴려보는 거예요. v1 → v2 → v3로 이어지는 패턴이 손에 익으면, 강의를 끝낸 뒤 본인 일에 만든 스킬도 같은 방식으로 다듬을 수 있습니다.

전과 후의 차이는 이렇습니다.

- 같은 김밥 주제로 v1과 풀스펙을 같은 폴더에 펼쳐놓고 본인 눈으로 차이를 봅니다. 출처 수, 멀티에이전트, 인터랙티브 질문 같은 단계가 시각적으로 드러나요.
- 본인 SKILL.md를 직접 텍스트 에디터로 열어 한두 줄 손봅니다. Part 5에서 설정한 trash-guard hook이 자동으로 백업해줘서, 잘못 만져도 복원이 됩니다.
- 같은 주제로 v2를 호출해 Before/After를 한 화면에서 비교합니다. 5분짜리 디벨롭의 효과가 명확히 보여요.

이번 클립에서 할 일은 다섯 단계입니다. 폴더에 두 결과 모아놓기 → 본인 SKILL.md 구조 분석 → 5관점 비교 → SKILL.md 손으로 v2 작성 → v2 호출.

## 핵심 개념

### "라이프사이클" — Clip 2가 만들기 전 사이클이면, Clip 3는 만든 후 사이클

스킬은 v1 한 번 만들고 끝이 아닙니다. 처음 만든 v1에는 아직 부족한 부분이 남아 있어서, 한 번 굴려보면 보강할 단계가 드러나요. 그 부족한 부분을 SKILL.md에 손으로 추가하면 v2가 됩니다.

```text
[Clip 2 메타 흐름]                 [Clip 3 라이프사이클]
                                  
워크플로우 잡기                    v1 ↔ 풀스펙 폴더에 모아놓기
   ↓                                 ↓
보완 + 단계 정의                   본인 SKILL.md 구조 분석
   ↓                                 ↓
SKILL.md 작성 (v1)                 5관점 비교
   ↓                                 ↓
호출 테스트                        SKILL.md 손으로 디벨롭 (v2)
                                     ↓
                                  v2 호출 + Before/After
```

같은 4단계 아니냐 싶을 수 있지만 결이 다릅니다. Clip 2는 "내가 모르는 단계까지 끌어오는" 사이클이고, Clip 3는 "이미 있는 것을 개선하는" 사이클입니다. 둘 다 손에 익혀두면 본인 일에 만든 스킬도 시간이 지나면서 자연스럽게 진화합니다.

### 별도 폴더 패턴 — 같은 폴더에 두 결과 모아놓기

비교를 손쉽게 하려면 두 결과를 한 폴더에 펼쳐놓는 게 가장 빠릅니다. 폴더 구조는 이렇게 갑니다.

```text
~/fastcampus-cc/50-my-work/Part06-스킬만들기/
└── 실습24-스킬분석개선/
    ├── 본인-v1-결과/
    │   └── 김밥맛집-{날짜}.md          ← Clip 2 산출이 자동 복사됨
    ├── 풀스펙-참고-결과/
    │   └── 전국_김밥_맛집_{날짜}/      ← 풀스펙 deep-research 호출 결과
    ├── 구조-분석.md
    ├── 비교-분석.md (5관점)
    ├── deep-research-v2-SKILL.md       ← v2 백업 사본
    └── 김밥맛집-v2-{날짜}.md           ← v2 호출 결과
```

`/part06` Clip 3 자동 셋업이 위 폴더와 Clip 2 산출 복사까지 처리해줍니다. 본인이 직접 `cp` 같은 명령을 입력하지 않아도 됩니다.

### 5관점 비교 — 출처 / 멀티에이전트 / 세션 폴더 / 산출 형식 / 인터랙티브

본인 v1과 풀스펙 결과를 같은 폴더에서 본 다음, 다섯 가지 관점에서 차이를 짚습니다.

| 관점 | 본인 v1 | 풀스펙 (예시) |
|---|---|---|
| 출처 수 | 5~10개 | 80개+ |
| 멀티에이전트 | 단일 호출 | 키워드별 병렬 |
| 세션 폴더 | 단일 마크다운 1장 | 날짜별 세션 폴더 + 메타 |
| 산출 형식 | 마크다운 | 마크다운 + HTML 리포트 |
| 인터랙티브 질문 | 없음 | 호출 시 옵션 질문 |

모든 5관점을 본인 v2에 다 적용할 필요는 없습니다. **본인 일에 가장 영향이 큰 1~2가지만** 골라 추가하세요. 풀스펙을 그대로 베끼는 게 목표가 아닙니다. 본인이 다음에 굴릴 때 부족했던 단계를 채우는 게 목표예요.

### 풀스펙은 어디서 가져오나 — gptaku-plugins 마켓플레이스

비교 대상이 되는 풀스펙 `deep-research` 스킬은 `gptaku-plugins` 마켓플레이스에서 받습니다. 외부 플러그인 모음이라 한 번 등록해두면 다음 클립에서도 같이 씁니다.

```text
/plugin marketplace add fivetaku/gptaku_plugins
/plugin install deep-research@gptaku-plugins
```

설치 후 클로드코드 재시작하면 `/deep-research` 슬래시 커맨드로 풀스펙 스킬을 호출할 수 있어요.

## 진행 흐름

### 1. `/part06` 호출 → Clip 3 선택

강의 워크스페이스에서 클로드코드를 시작하고 Clip 3를 선택합니다.

```text
/part06
```

자동 셋업이 다음을 처리해줍니다.

```text
✓ clip-02 deep-research v1 확인됨
✓ 실습23 v1 산출 1개 → 본인-v1-결과/ 복사 (원본 보존)
✓ 풀스펙 deep-research 플러그인 확인됨 — 비교 대상
✓ trash-guard hook 확인됨
✓ 진도 폴더 + 비교 폴더 패턴 (본인-v1-결과 / 풀스펙-참고-결과) 준비 완료
```

풀스펙 플러그인이 미설치 상태면 안내가 뜹니다. 한 줄 명령으로 설치 후 다시 `/part06` Clip 3를 호출하세요.

### 2. STEP 1 — 풀스펙 결과를 같은 폴더에 모아놓기

풀스펙 deep-research를 호출하면서 출력 경로를 같은 비교 폴더로 지정합니다.

```text
전국 김밥 맛집 주제로 딥리서치 해줘.
결과는 ~/fastcampus-cc/50-my-work/Part06-스킬만들기/실습24-스킬분석개선/풀스펙-참고-결과/
에 저장해줘.
```

5~10분 정도 걸려요. 진행 중에 어떤 단계가 풀스펙에서 실행되는지 화면을 한 번 봐두세요. 키워드 분해, 병렬 출처 수집, 인터랙티브 질문 같은 단계가 보입니다.

완료 후 안티그래비티에서 비교 폴더를 열면 두 결과가 나란히 보입니다.

```text
실습24-스킬분석개선/
├── 본인-v1-결과/김밥맛집-{날짜}.md
└── 풀스펙-참고-결과/전국_김밥_맛집_{날짜}/
```

### 3. STEP 2 — 본인 SKILL.md 구조 분석

본인이 Clip 2에서 만든 `~/fastcampus-cc/.claude/skills/deep-research/SKILL.md`를 클로드코드한테 분석시킵니다.

```text
내가 만든 ~/fastcampus-cc/.claude/skills/deep-research/SKILL.md
구조를 분석해서 약점을 지적해줘.
결과는 ~/fastcampus-cc/50-my-work/Part06-스킬만들기/실습24-스킬분석개선/구조-분석.md
에 저장해줘.
```

`구조-분석.md`에는 description, 워크플로우 단계 수, 결과물 저장 규칙 같은 항목별 평가가 한 장으로 정리됩니다.

### 4. STEP 3 — 5관점 비교

이제 v1 결과와 풀스펙 결과를 5관점에서 비교합니다.

```text
본인-v1-결과/김밥맛집-{날짜}.md 와
풀스펙-참고-결과/전국_김밥_맛집_{날짜}/ 산출물을
다섯 관점에서 비교해줘.

관점:
1) 출처 수와 출처 다양성
2) 멀티에이전트 / 병렬 처리 여부
3) 세션 폴더 vs 단일 파일
4) 산출 형식 (마크다운만? HTML 추가?)
5) 호출 시 인터랙티브 질문 여부

결과는 비교-분석.md 에 표 형식으로 저장.
```

`비교-분석.md`를 보면 본인 v1에서 어느 칸이 비어 있는지 한눈에 들어와요. 다섯 칸 중 부족한 게 3~4개일 텐데, 다 채우려고 하지 마세요. **본인 일에 가장 영향이 큰 1~2가지만** 다음 단계로 가져갑니다.

### 5. STEP 4 — SKILL.md 손으로 디벨롭 → v2

가져갈 개선 1~2가지를 정했으면, 클로드코드한테 SKILL.md를 손봐달라고 요청합니다.

```text
~/fastcampus-cc/.claude/skills/deep-research/SKILL.md를 다음 두 가지 개선과 함께 v2로 디벨롭해줘.

1) 호출 시 인터랙티브 질문 추가
   - 정렬 기준 (최신순/관련도순)
   - 출처 가중치 (한국 블로그 위주 / 글로벌 위주 / 둘 다)

2) 결과물에 출처 카테고리 헤더 추가
   - 블로그 / 뉴스 / SNS / 커뮤니티 별로 묶기

수정 전에 백업 떠두기 — trash-guard hook이 자동으로 떠줘.
디벨롭한 SKILL.md 사본을 ~/fastcampus-cc/50-my-work/Part06-스킬만들기/실습24-스킬분석개선/deep-research-v2-SKILL.md 에 저장.
```

trash-guard hook이 SKILL.md를 수정하기 전에 자동으로 `.bak.YYYYMMDD-HHMMSS` 백업을 떠줍니다. 만약 v2가 더 안 좋아지면 백업으로 되돌리면 돼요.

### 6. STEP 5 — v2 호출 + Before/After

스킬이 수정됐으니 클로드코드를 재시작한 뒤 같은 주제로 v2를 호출합니다.

```bash
exit
claude
```

```text
전국 김밥 맛집에 대해 깊이 리서치해줘
```

이번엔 인터랙티브 질문이 먼저 뜨고, 답변에 따라 결과물이 달라집니다. 출처 카테고리 헤더도 들어가요.

결과를 `실습24-스킬분석개선/김밥맛집-v2-{날짜}.md`로 저장하고, Clip 2의 v1 결과(`본인-v1-결과/김밥맛집-{날짜}.md`)와 같은 화면에 펼쳐서 차이를 봅니다.

5분짜리 디벨롭으로 결과물의 깊이가 얼마나 바뀌는지 한눈에 들어와요. 이게 라이프사이클의 첫 한 바퀴입니다.

| 백업 파일 안 보임 | trash-guard hook 미설정 | Part 5 Clip 4에서 만든 hook 확인. 없으면 수동 `cp SKILL.md SKILL.md.bak` |
| 비교 분석이 표가 아니라 문장으로 나옴 | "표 형식" 명시 안 됨 | "마크다운 표로 저장해줘" 재요청 |
| 5관점 다 채우려고 무리함 | 풀스펙 따라잡기 욕심 | 1~2가지만. 본인 일에 영향 클 단계 우선 |
| v2가 v1보다 안 좋아짐 | 너무 많이 수정 | 백업(`SKILL.md.bak.*`)으로 롤백 후 1가지만 다시 적용 |
