## 이 클립에서 만드는 것

본인 손으로 처음 만드는 스킬이에요. 이름은 `deep-research`. 어떤 주제를 입력하면 여러 출처에서 자료를 모아 한 장으로 정리해주는 워크플로우입니다.

다만 결과물만 보고 끝나면 의미가 적어요. **메타 흐름 4단계**를 손으로 한 바퀴 굴려보는 게 진짜 목적입니다. Part 6의 9개 실습이 모두 같은 4단계를 반복하므로, Clip 2에서 손에 익혀두면 나머지 8개는 같은 박자로 따라옵니다.

전과 후의 차이는 이렇습니다.

- 새 스킬을 만들 때 "딥리서치 스킬 만들어줘" 같은 명령형 대신, "워크플로우를 어떻게 구성해야 할까?"로 시작하는 습관이 생깁니다.
- AI에게 맡길 단계와 본인이 정해야 할 단계를 구분하는 감이 잡힙니다.
- 강의 워크스페이스 안에 본인 자산 스킬 폴더 하나가 처음 생깁니다 — 이름은 `deep-research`.

이번 클립에서 할 일은 네 단계입니다. 워크플로우 조사 → 단계 보완 → SKILL.md 작성 → 호출 테스트. 마지막에 김밥 맛집 주제로 호출해서 결과까지 확인합니다.

## 핵심 개념

### 메타 흐름 4단계 — 왜 "그냥 만들어줘"가 안 통하나

스킬을 만들 때 첫 프롬프트를 어떻게 던지느냐에 따라 결과의 깊이가 갈립니다. 두 가지를 비교해 보세요.

```text
나쁜 시작
"딥리서치 스킬 만들어줘"
→ 클로드코드가 곧장 SKILL.md를 쓰기 시작
→ 본인이 그동안 알고 있던 "딥리서치"의 한계가 그대로 반영됨

좋은 시작
"어떤 주제에 대해 깊이 조사해서 정리하려는데,
 어떻게 워크플로우를 구성해야 할까?"
→ 클로드코드가 먼저 "딥리서치를 어떻게 하는지" 그 자체를 조사
→ 다관점 출처 수집, 교차 검증, 종합, 출처 명시 같은 단계가 펼쳐짐
→ 그중에 본인이 모르던 단계가 들어옴
```

차이는 한 문장이지만 결과는 크게 갈립니다. 좋은 시작은 본인 머릿속 한계를 넘어서는 단계를 끌어와줘요. 그래서 이번 클립부터 9개 실습 모두 같은 4단계를 따릅니다.

```text
[1단계] 워크플로우 잡기 (방법론 조사)
   "[하려는 일] 만들려는데, 어떻게 워크플로우를 구성해야 할까?"

[2단계] 보완 + 실행 가능 단계 정의
   정리된 단계를 같이 살펴보고 보완
   → AI에게 맡길 단계와 코드로 분리할 단계를 여기서 구분

[3단계] 스킬로 만들기
   정의된 흐름을 SKILL.md로 정리

[4단계] 만든 스킬로 테스트·검증
   실제로 호출해서 description 매칭과 결과 확인 + 부족한 부분 손보기
```

이 4단계가 손에 익으면 Part 6을 마친 뒤에도 본인 일에 새 스킬을 만들 때 같은 박자로 시작할 수 있습니다. SKILL.md 작성 자체는 클로드코드가 합니다. 본인이 챙겨야 할 건 시작 프롬프트뿐이에요.

### 스킬 폴더 구조 — `~/fastcampus-cc/.claude/skills/`

만들어진 스킬은 강의 워크스페이스 안에 폴더 한 장으로 떨어집니다.

```text
~/fastcampus-cc/.claude/skills/
└── deep-research/
    └── SKILL.md         ← 본문 (description + 워크플로우 정의)
```

`SKILL.md` 맨 위에 description 한 줄이 있어야 클로드코드가 호출 시 매칭합니다. "딥리서치", "깊이 조사", "여러 출처로 종합" 같은 자연어 키워드를 description에 넉넉히 넣어두면, 사용자가 어떤 표현으로 부르든 같은 스킬이 발동돼요.

### v1만 만든다 — 완성은 다음 클립에서

오늘 만들 결과물은 본인 v1입니다. 미리 만들어둔 풀스펙 스킬과는 아직 차이가 큽니다. 출처가 부족할 수도 있고, 인터랙티브 질문이 빠질 수도 있어요.

괜찮습니다. **본인 v1을 한 번 만들어야** Clip 3에서 풀스펙과 비교했을 때 본인이 놓친 단계가 보입니다. 처음부터 완벽하게 만들려고 하지 마세요.

## 진행 흐름

### 1. `/part06` 호출 → Clip 2 선택

강의 워크스페이스에서 클로드코드를 켜고 시작합니다.

```text
/part06
```

스킬이 진도를 확인하고 클립 선택지를 띄웁니다. Clip 2를 선택하면 작업 폴더가 자동으로 준비돼요.

```text
✓ ~/fastcampus-cc/50-my-work/Part06-스킬만들기/실습23-딥리서치스킬/ 준비 완료
✓ ~/fastcampus-cc/.claude/skills/ 준비 완료
```

진도 폴더는 결과물 기록용이고, `.claude/skills/`는 만든 스킬이 활성화될 자리입니다. 두 폴더가 분리돼 있는 이유는 [Clip 1의 결과물 저장 규칙](#part-6-01-skill-intro)에서 짚었던 그대로예요.

### 2. STEP 1 — 워크플로우 잡기

클로드코드 세션에서 명령형 대신 의문문으로 시작합니다.

```text
어떤 주제에 대해 깊이 조사해서 정리하려는데,
어떻게 워크플로우를 구성해야 할까?
```

클로드코드가 "딥리서치를 어떻게 하는지"를 직접 조사합니다. 1~2분 정도 걸려요. 정리된 단계는 보통 이런 모습입니다.

```text
1. 주제 키워드 분해 (하위 키워드 — 지역·종류·후기·가격·트렌드 등)
2. 다관점 출처 수집 (블로그·SNS·뉴스)
3. 교차 검증 (충돌 정보 식별)
4. 종합 (핵심 한 줄 + 카테고리별 정리 + 한계 명시)
5. 한국어 정리 + 출처 명시
```

조사 결과에 따라 단계 수와 내용은 조금씩 달라집니다. 5단계가 7단계가 될 수도 있고, 항목 이름이 바뀔 수도 있어요. 그게 정상입니다.

### 3. STEP 2 — 보완 + 실행 가능 단계 정의

정리된 단계를 보면 한두 개는 개선할 부분이 보입니다. 예를 들어 "교차 검증"이 너무 모호하거나, "출처 명시" 단계가 빠져 있을 수 있어요.

```text
정리된 단계들 보니까 좋은데, 두 가지만 보완하면 좋겠어.
1) 한국 사이트 가중치(블로그·디시·네이버 카페) 추가
2) 모은 자료에서 충돌 정보 발견 시 어떻게 처리할지 명시
```

클로드코드가 단계를 다듬어줍니다. 이때 **AI에게 맡길 단계와 본인이 정해야 할 단계를 구분**해두면 됩니다.

- AI에게 맡길 단계: 자료 수집, 종합, 정리 — 매번 다른 결과가 나와도 OK인 부분
- 본인이 정해야 할 단계: 키워드 분해 방식, 출처 가중치, 출력 형식 — 매번 같은 규칙으로 동작해야 하는 부분

Clip 2에서는 모든 단계를 SKILL.md 본문에 자연어로 적습니다. 결정론 코드 분리(스크립트로 빼는 패턴)는 CH02 Clip 5부터 제대로 다뤄요.

### 4. STEP 3 — SKILL.md로 스킬화

이제 클로드코드한테 SKILL.md 작성을 부탁합니다.

```text
지금 정의한 워크플로우를 SKILL.md로 정리해줘.
- 위치: ~/fastcampus-cc/.claude/skills/deep-research/SKILL.md
- description에 "딥리서치", "깊이 조사", "심층 조사", "여러 출처 종합" 키워드 넣기
- 워크플로우 단계를 그대로 본문에 옮기기
- 한국어로 답변 + 결과물은 ~/fastcampus-cc/50-my-work/Part06-스킬만들기/실습23-딥리서치스킬/{주제}-{날짜}.md 에 저장
```

저장이 끝나면 안티그래비티 에디터에서 새로 생긴 `SKILL.md`를 한 번 열어보세요. description 한 줄과 본문에 단계가 그대로 들어가 있어야 합니다.

스킬은 클로드코드 세션 시작 시 로드돼요. 새 스킬을 만든 직후엔 `/exit` 후 다시 `claude`로 켜야 인식됩니다.

```bash
exit
claude
```

### 5. STEP 4 — 만든 스킬로 테스트·검증

재시작한 세션에서 자연어로 호출합니다.

```text
전국 김밥 맛집에 대해 깊이 리서치해줘
```

description이 매칭되면 `deep-research` 스킬이 발동되면서 SKILL.md에 적힌 단계대로 작업이 시작됩니다. 진행 중에 클로드코드가 어떤 단계를 어떻게 수행하는지 보세요.

결과는 두 파일로 떨어집니다.

- `실습23-딥리서치스킬/김밥맛집-{날짜}.md` — 본인 v1 스킬의 산출물
- `실습23-딥리서치스킬/김밥-검증.md` — 호출이 잘 됐는지 확인 메모 (직접 작성)

검증 메모에는 두 가지를 적어두세요.

```text
1. description 매칭 확인 — "전국 김밥 맛집" 입력에 deep-research가 자동 발동됐나?
2. 결과물 만족도 — 어떤 단계가 약했나? 출처가 적었나? 정리가 거칠었나?
```

이 메모가 Clip 3의 비교·개선 사이클에서 출발점이 됩니다.

### 6. 본인 v1의 한계 인식 + Clip 3 예고

만든 결과물을 보면 부족한 부분이 보일 거예요. 예를 들면 이런 것들.

- 출처가 5~10개 정도로 적다 — 풀스펙 스킬은 80개+ 출처를 모음
- 멀티에이전트 패턴이 없다 — 한 번에 한 가지만 조사
- 결과물이 단일 마크다운 한 장 — 풀스펙은 세션 폴더 + 인터랙티브 질문 포함

괜찮습니다. **본인 v1의 한계를 직접 본 게 오늘의 핵심 수확**입니다. Clip 3에서는 이 v1과 미리 만들어둔 풀스펙 스킬을 같은 폴더에 모아놓고 5관점에서 비교합니다. 본인이 놓친 단계가 시각적으로 드러나요.
