# Part 6 / Clip 4: skillers-suda — 스킬 만들기 자체를 자동화한 도구

> 강의 영상: Part 6 / Ch 01 / Clip 4 (~20분)
> 만드는 것: skillers-suda 도구로 본인 두 번째 스킬 자동 생성 (이름은 본인 아이디어에 따라 가변)
> 도구: `gptaku-plugins` 마켓플레이스의 `skillers-suda` 플러그인

---

## 이 클립에서 만드는 것

Clip 2에서는 메타 흐름 4단계를 손으로 굴려서 `deep-research` v1을 만들었어요. 이번엔 그 4단계를 인터뷰 흐름으로 자동화한 도구를 써봅니다. 이름은 `skillers-suda` — 네 명의 전문가 AI가 아이디어 분석가·스킬 작가·검증자·통합자 역할로 수다를 떨면서 `SKILL.md`를 자동 생성해줍니다.

이 클립은 두 부분으로 갈라집니다.

- **1부 도구 설명** — `skillers-suda`가 어떻게 동작하는지, 왜 만들어졌는지, 어떤 상황에 적합한지 살펴봅니다.
- **2부 활용** — 본인 업무에 가까운 주제 한 줄을 정해서, `skillers-suda`로 두 번째 스킬을 직접 생성합니다.

2부에서 어떤 스킬을 만들지는 본인이 정합니다. 예를 들어 영업 일을 하는 분이라면 "고객사 미팅 전 사전 조사 정리", 마케팅 일을 하는 분이라면 "경쟁사 신제품 모니터링" 같은 식이에요. 김밥 맛집 같은 시연 주제 말고, 본인 일에 바로 가져갈 만한 걸 골라야 마지막에 호출했을 때 가치가 보입니다.

이번 클립에서 할 일은 여섯 단계입니다. 1부 도구 설명 → 본인 아이디어 정하기 → skillers-suda 호출 → 인터뷰 답변 → 자동 생성된 스킬 호출 → 다듬기.

---

## 핵심 개념

### 정석 vs 자동 — 두 방식의 비교

같은 일을 두 가지 방법으로 할 수 있을 때, 둘 다 익혀두면 상황에 따라 골라 쓸 수 있어요. Clip 2의 정석 방식과 Clip 4의 자동 방식이 그런 관계입니다.

| 항목 | Clip 2 정석 | skillers-suda |
|---|---|---|
| 작성 방식 | 본인이 SKILL.md를 직접 짜 나감 | 4명 전문가 AI 인터뷰로 자동 생성 |
| 소요 시간 | 20분 | 5~7분 |
| 구조 이해 | 손으로 짜봐서 단계별 의도가 잡힘 | 블랙박스에 가까움 |
| 본인 일 응용 | 한 번 익히면 평생 쓸 수 있음 | 새 아이디어가 떠올랐을 때 빠른 첫 동작 |
| 추천 시점 | 처음 1~2개 스킬 만들 때 | 손에 메타 흐름이 익은 뒤, 빠르게 원형을 만들고 싶을 때 |

Clip 2 정석은 구조를 이해하는 용도, Clip 4 skillers-suda는 평생 쓰는 도구입니다. 새 스킬 아이디어가 떠오를 때마다 손으로 SKILL.md를 짜기엔 부담이 크니까, 메타 흐름을 한 번 익힌 뒤로는 이 도구를 부르는 게 일상 패턴이 됩니다.

### 4명 전문가 AI — 누가 무엇을 하나

`skillers-suda`를 호출하면 한 명의 AI가 답하는 게 아니라 네 명의 역할이 순서대로 나옵니다.

```text
[아이디어 분석가]  → 본인 한 줄 아이디어 듣고 "정확히 뭘 하고 싶은가?" 질문
       ↓
[스킬 작가]       → 분석된 의도 기반으로 워크플로우 단계 초안
       ↓
[검증자]         → 단계에 빠진 게 있는지, 모호한 표현이 있는지 검토
       ↓
[통합자]         → 최종 SKILL.md 형태로 묶어 저장
```

각 단계마다 본인에게 한두 가지 짧은 질문이 돌아옵니다. 답이 길 필요 없어요. 한 줄씩 답하면서 자연스럽게 메타 흐름이 인터뷰 형태로 흘러갑니다.

### 자동 생성된 스킬도 다듬어야 한다 — Clip 3 사이클 재사용

`skillers-suda`가 만든 SKILL.md가 본인 일에 100% 맞지는 않아요. 처음 호출해서 결과를 본 다음, 부족하거나 본인 일과 안 맞는 부분 1~2가지를 손으로 손봅니다. 이 단계가 [Clip 3 — 라이프사이클](#part-6-03-evaluate-improve)에서 익힌 비교·개선 사이클의 재적용이에요.

자동 생성 → 다듬기로 이어지는 흐름이 손에 익으면, 본인 일에 새 스킬 만들 때마다 같은 박자로 굴릴 수 있습니다.

## 진행 흐름

### 1. `/part06` 호출 → Clip 4 선택

강의 워크스페이스에서 클로드코드를 시작합니다.

```text
/part06
```

자동 셋업이 `skillers-suda` 플러그인 설치 여부를 확인합니다.

```text
✓ skillers-suda 플러그인 확인됨
✓ 진도 폴더 준비 완료
```

미설치 상태면 안내가 뜹니다. 한 줄 명령으로 설치한 뒤 다시 진행하세요.

```text
/plugin marketplace add fivetaku/gptaku_plugins
/plugin install skillers-suda@gptaku-plugins
```

설치 후 클로드코드 재시작이 필요합니다.

### 2. STEP 1 — 1부 도구 설명 (~2분)

스킬이 처음에 1부 도구 설명을 한 화면으로 띄워줍니다. 위의 "정석 vs 자동" 표와 "4명 전문가 AI" 흐름이 한 번에 보입니다.

여기서 본인에게 맞을지 판단해보세요. 만약 새 스킬 아이디어가 아직 없다면, Clip 4는 일단 도구 설명만 보고 넘어가도 됩니다. 본인 아이디어가 떠올랐을 때 다시 호출해도 결과는 같아요.

### 3. STEP 2 — 본인 아이디어 한 줄 정하기 (~1.5분)

어떤 스킬을 만들지 정합니다. 한 줄로 짧게 적어두면 좋아요. 예시.

```text
- 마케팅: "경쟁사 신제품 모니터링 — 매주 출시 정보 정리"
- 영업: "고객사 미팅 전 사전 조사 — 회사 동향 + 최근 보도"
- 운영: "주간 보고용 데이터 요약 — 지표 5개 한 페이지"
- 학습: "관심 분야 새 논문 정리 — 매월 핵심 5편"
```

본인 일에 가장 자주 반복하는 작업 중 하나를 고르세요. 머릿속에 막연한 아이디어 상태로 적어도 괜찮습니다. 다음 단계에서 `skillers-suda`가 구체화를 도와줍니다.

### 4. STEP 3 — skillers-suda 호출 + 4명 인터뷰 시작

자연어로 호출합니다.

```text
경쟁사 신제품 모니터링 스킬 만들고 싶어. skillers-suda 호출.
```

또는 슬래시 커맨드로.

```text
/skillers-suda
```

호출되면 첫 번째 역할인 **아이디어 분석가**가 나와서 한두 가지 질문을 던집니다. 예를 들면.

```text
[아이디어 분석가]
"경쟁사 신제품 모니터링"에서 두 가지 확인할게요.

1) 경쟁사는 미리 정해진 회사 목록입니까, 매번 자유 입력입니까?
2) 모니터링 결과는 어떻게 받고 싶으세요? (이메일 / 슬랙 / 마크다운 파일)
```

한 줄씩 답하면 됩니다.

### 5. STEP 4 — 인터뷰 답변 + 자동 생성

답변을 이어가면 **스킬 작가** → **검증자** → **통합자** 순서로 역할이 바뀝니다. 5~7분 동안 두세 번 정도 짧은 질문이 더 돌아오고, 마지막에 `SKILL.md`가 자동으로 저장돼요.

```text
✓ ~/fastcampus-cc/.claude/skills/competitor-monitor/SKILL.md 생성 완료
✓ 실습25/skill-source/SKILL.md 사본 저장
```

스킬 이름은 인터뷰 답변에 따라 자동으로 정해집니다. 영문 kebab-case로 떨어져요. 마음에 안 들면 통합자 단계에서 "이름을 ~로 바꿔줘"라고 요청할 수 있습니다.

### 6. STEP 5 — 첫 호출 시연

스킬이 새로 생겼으니 클로드코드를 재시작한 뒤 자연어로 호출합니다.

```bash
exit
claude
```

```text
이번 주 경쟁사 신제품 모니터링 돌려줘
```

description이 매칭되면 새 스킬이 발동되면서 SKILL.md에 적힌 단계대로 작업이 시작됩니다. 5~7분짜리 인터뷰로 만들어진 스킬이 실제 결과물까지 한 번에 굴러가는 걸 한 화면에서 보세요.

### 7. STEP 6 — 다듬기 (Clip 3 사이클 재사용)

첫 호출 결과를 보고 부족한 부분 1~2가지를 손으로 손봅니다. SKILL.md를 직접 열어 한두 줄 추가하거나, 클로드코드한테 부탁해도 됩니다.

```text
방금 만든 competitor-monitor SKILL.md에서 출처 카테고리를
"공식 보도 / 외부 매체 / 커뮤니티"로 분리하는 단계 추가해줘.
```

Part 5에서 설정한 trash-guard hook이 SKILL.md 백업을 자동으로 떠줘서 잘못 만져도 복원이 됩니다. [Clip 3 — 라이프사이클](#part-6-03-evaluate-improve)에서 익힌 비교·개선 사이클을 그대로 한 번 더 굴리는 거예요.

## 결과물

이번 클립이 끝나면 아래가 남습니다.

| 결과물 | 위치 | 설명 |
|---|---|---|
| `SKILL.md` | `~/fastcampus-cc/.claude/skills/{생성된-스킬명}/SKILL.md` | skillers-suda가 자동 생성한 본인 두 번째 스킬 |
| `skill-source/SKILL.md` | `실습25-스킬러들의수다/` | 진도 폴더 사본 |
| 첫 호출 결과 | `실습25-스킬러들의수다/` | 생성된 스킬을 처음 굴려본 산출물 |
| `README.md` | `실습25-스킬러들의수다/` | 자동 생성된 4단계 + 본인 다듬기 1~2건 기록 |

`완료` 또는 `/wrap`을 입력하면 `progress.json`의 `practice_completed`에 `"실습 25"`가 추가되고, `skills_created`에 생성된 스킬명이 들어갑니다. 이름은 본인 인터뷰 답변에 따라 가변이라 미리 정해져 있지 않아요.

## 자주 막히는 포인트

| 증상 | 원인 | 해결 |
|---|---|---|
| `skillers-suda 플러그인 확인되지 않음` | gptaku-plugins 마켓플레이스 미설치 | `/plugin marketplace add fivetaku/gptaku_plugins` 후 `/plugin install skillers-suda@gptaku-plugins` |
| 인터뷰 질문이 너무 많이 옴 | 본인 아이디어가 모호함 | "한 번에 한 질문씩만, 5개까지" 추가 요청 |
| 인터뷰 질문이 1~2개만 오고 끝남 | skillers-suda가 빨리 결론 내려고 함 | "단계별로 더 깊게 물어봐줘" 재요청 |
| 자동 생성된 SKILL.md가 너무 일반적 | 본인 답변이 추상적 | 답변에 구체 도메인 단어 (회사명, 지표명) 한두 개씩 넣기 |
| 스킬 이름이 마음에 안 듦 | 자동 명명 | 통합자 단계에서 "이름을 ~로 바꿔줘" 또는 SKILL.md 폴더 이름 변경 |
| 새 스킬이 호출 안 됨 | 재시작 안 함 | `exit` 후 `claude`로 재시작 |
| description 매칭이 약함 | 자동 생성 description이 좁음 | SKILL.md 열어 description 키워드 4~5개로 확장 |
| Clip 2 정석 방식과 혼동됨 | 둘 다 같은 결과를 만드는 것이 아닌지 의문 | 정석은 구조 이해용, 자동은 일상 사용용. 둘 다 익혀두면 상황에 따라 선택 |
| 본인 일에 안 맞는 스킬이 만들어짐 | 아이디어가 너무 막연 | STEP 2로 돌아가 한 줄 아이디어를 더 구체화 후 재호출 |
